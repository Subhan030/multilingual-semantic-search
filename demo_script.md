# 🎬 Multilingual Semantic Search Backend Walkthrough

**Target Length:** ~2 minutes
**Tone:** Informative, technical but accessible, enthusiastic

---

### [0:00 - 0:15] Hook & Introduction
* **Visual:** Wait on the landing page of the Streamlit web interface with the title "🌐 Multilingual Semantic Search".
* **Audio:** "Hi everyone! Today I want to show you my Multilingual Semantic Search and RAG application. Unlike standard search bars that rely on exact keywords, this engine actually understands the *meaning* behind your questions and can seamlessly search across different languages. Let me show you how the magic works under the hood!"

### [0:15 - 0:45] Indexing the Document (Chunking & Embedding)
* **Visual:** Check the "Show sample document" box to reveal the Hindi Corporate text. Then click the **"Load sample document"** button. A green success message appears indicating 16 chunks were indexed.
* **Audio:** "First, we need to load some data. Here I have a Corporate Benefits guide written entirely in Hindi. When I hit 'Load', my FastAPI backend receives the text and strategically slices it into dense **70-word chunks**, maintaining a **15-word overlap** between them so sentences don't forcefully skip context. 
Then, I run each chunk through a specialized Sentence-Transformer model. This generates mathematical vectors—known as embeddings—that capture the true semantic intent. Finally, these vectors are aggressively normalized and inserted into an in-memory **FAISS Index**, creating our lightning-fast, searchable vector database!"

### [0:45 - 1:20] Cross-Lingual Search Execution (Vector Similarity)
* **Visual:** *Ensure the RAG checkbox is UNCHECKED for now.* Type an English query into the box, such as: `How much paid time off do I get?` or `Is there a gym stipend?`. Click **Search**. Show the Hindi paragraphs popping up natively in the UI with their similarity scores.
* **Audio:** "Now for the fun part: Cross-lingual retrieval. Even though the document is completely in Hindi, I'm going to type my query in English. When I hit search, the system converts my English question into its own vector and executes a highly-optimized **Cosine Similarity** search against the FAISS index. Look at that! It perfectly retrieved the exact Hindi chunks discussing 'Paid Time Off' and 'Vacations'. The engine isn't translating keywords; it fundamentally understands that the English question and the Hindi paragraph share the exact same geometric space."

### [1:20 - 1:50] Synthesizing Answers with Real-Time RAG
* **Visual:** Now, click the **"Answer using retrieved context (RAG)"** checkbox. Leave the same query. Click **Search** again. Highlight the natural English answer generated by the AI before scrolling down to the Hindi context.
* **Audio:** "But how do we make this readable for the user? This is where Retrieval-Augmented Generation, or RAG, steps in. Let's toggle the RAG feature and search again. This time, my backend pulls those relevant Hindi chunks, bundles them as strict context, and fires a prompt off to Groq's Llama-3 model. Acting as both translator and synthesizer, the LLM reads the Hindi context and instantly generates a perfectly grounded, conversational English answer based *only* on the data we provided."

### [1:50 - 2:00] Outro
* **Visual:** Scroll freely through the UI showcasing the speed, or return to the main upload mechanism. 
* **Audio:** "And that is how we build a true knowledge engine! We combined fine-tuned chunking, dense vector similarity, and real-time RAG inference into one scalable Streamlit and FastAPI stack. Thanks for watching!"
